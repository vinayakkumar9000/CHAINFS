# ChainFS Setup Guide

This guide walks you through every step needed to run ChainFS from a fresh clone.

---

## Prerequisites

| Tool | Minimum version | Notes |
|---|---|---|
| Node.js | **22.5.0** | Required for the built-in `node:sqlite` module |
| npm | 10+ | Comes with Node.js |
| Python | **3.9** | Required for `chaincli` and Python bots |
| pip | 23+ | Comes with Python |
| Git | any | For cloning |

Check your versions:

```bash
node --version   # must be ≥ 22.5.0
python3 --version
```

---

## 1 — Clone the repository

```bash
git clone https://github.com/<your-org>/CHAINFS.git
cd CHAINFS
```

---

## 2 — Install Node.js dependencies

```bash
npm install
```

This installs:
- `ethers` — JSON-RPC / contract interaction
- `commander` — CLI argument parsing
- `hardhat` and `@nomicfoundation/hardhat-toolbox` — testing and deployment

---

## 3 — Install Python dependencies

```bash
pip install -r requirements.txt
```

This installs:
- `web3` — Web3.py JSON-RPC library
- `eth-account` — wallet / signing support

---

## 4 — Compile the smart contract

```bash
npm run compile
```

This generates `artifacts/contracts/ChainFS.sol/ChainFS.json`, which contains the
ABI and bytecode required by the Python bots and deployment scripts.

---

## 5 — Configure environment

Copy the example environment file:

```bash
cp .env.example .env
```

Edit `.env` with your values:

```
CHAINFS_RPC_URL=http://127.0.0.1:8545
CHAINFS_CONTRACT=0x<your deployed contract address>
CHAINFS_PRIVATE_KEY=0x<your wallet private key>
CHAINFS_DB_PATH=~/.chainfs/chainfs.db
CHAINFS_INDEX_DB_PATH=./data/chainfs.db
```

> **Security:** Never commit `.env` or any file containing your private key to version
> control. Both `.env` and `config/wallet_private_key.txt` are already in `.gitignore`.

---

## 6 — Deploy the contract

### Option A — Interactive (recommended for testnet)

Use the Python `chaincli manager` command. It will:
1. Ask you to create a new wallet or import an existing private key
2. Check your balance and prompt you to use a faucet if empty
3. Deploy the ChainFS contract
4. Save the contract address to `config/contract.json` automatically

```bash
python chaincli manager
```

After deployment, `chaincli upload`, `chaincli download`, etc. will read the address
from `config/contract.json` automatically — no extra environment variables needed for
the Python CLI path.

### Option B — Hardhat local node (recommended for development)

```bash
# Terminal 1 — start a local Hardhat node
npx hardhat node

# Terminal 2 — deploy the contract
npm run deploy:local
# Output:
#   Deploying ChainFS with account: 0x…
#   ChainFS deployed to: 0xABC123…
#   Set CHAINFS_CONTRACT=0xABC123…

# Then set environment variables for the Node.js CLI
export CHAINFS_RPC_URL=http://127.0.0.1:8545
export CHAINFS_CONTRACT=0xABC123…
export CHAINFS_PRIVATE_KEY=0x<Hardhat account private key>
```

To use `chaincli` with a Hardhat deployment, also create the config file:

```bash
mkdir -p config
echo '{"contractAddress":"0xABC123…"}' > config/contract.json
```

### Option C — Public testnet deployment

```bash
# Set RPC URL and private key in .env, then:
npm run deploy
# → ChainFS deployed to: 0x…
```

Update `CHAINFS_CONTRACT` in your `.env` and `config/contract.json` with the printed
address.

## 7 — RPC Configuration

### Local Hardhat node

```
CHAINFS_RPC_URL=http://127.0.0.1:8545
```

### SKALE Base Sepolia Testnet (default for Python bots)

The Python bots (`manager_bot.py`, `storage_bot.py`) default to:

```
RPC_URL = https://base-sepolia-testnet.skalenodes.com/v1/jubilant-horrible-ancha
Chain ID: 324705682
```

SKALE uses a `CREDIT` token instead of ETH. Obtain test tokens from the faucet:
<https://base-sepolia-faucet.skale.space>

---

## 8 — Wallet Setup

### Using `chaincli manager` (interactive)

The interactive manager will generate a new wallet or accept an existing private key.
The private key is saved to `config/wallet_private_key.txt` (mode 600).

### Manual wallet setup (Node.js CLI)

Export the private key directly:

```bash
export CHAINFS_PRIVATE_KEY=0x<64-hex-char private key>
```

### Hardhat test accounts

When using a local Hardhat node, test accounts and their private keys are printed in
the Hardhat node output:

```
Account #0: 0xf39Fd6…  (10000 ETH)
Private Key: 0xac0974bec39a17e36ba4a6b4d238ff944bacb478cbed5efcae784d7bf4f2ff80
```

---

## 9 — Run tests

```bash
npm test
```

Tests use an in-memory Hardhat EVM node — no external RPC or funded wallet needed.

## Directory structure after setup

```
CHAINFS/
├── artifacts/          ← generated by npm run compile
│   └── contracts/ChainFS.sol/ChainFS.json
├── config/             ← created by chaincli manager
│   ├── contract.json
│   └── wallet_private_key.txt  (mode 600)
├── data/               ← created on first sync/upload
│   └── chainfs.db      (SQLite index DB)
├── downloads/          ← created by chaincli download
├── contracts/
├── src/
├── test/
├── .env
└── …
```
